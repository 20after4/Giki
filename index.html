[b]Welcome to GIKI[/b] [b]GIKI[/b] is a simple, real time, asynchronous parsing framework that suppors a wide variety of parsers. Currently, BBCode and wiki text are supported, and it is very easy to customize and create new parsers. The performance of the parsers is outstanding as they make use of webworkers via the [url=https://github.com/guyht/GWorker]GWorker[/url] library. [color=#FDD017]This text is parsed in real time. If you would like to try it out for yourself, simply click on the try me button on the right hand side of the page and make some edits. The edits will appear in real time. [/color] [b]Installation[/b] Simply add the following to your HEAD tag: [code]<script src="giki.js" type="text/javascript">[/code] If you would like the script to be compatible with older browsers that do not support web workers, then also download and add the gworker.js script to yoru HEAD tag. This is recommended. [code] <script src="gworker.js" type="text/javascript">[/code] NOTE: The gworker script must be included [i]before[/i] the giki script [b]Usage[/b] Add a text area to your page to receive input, and a div to received the parsed data and make sure they have unique ids. [code] <div id="giki_div"></div> <textarea id="giki_ta"></textarea> [/code] Then simply invoke the giki object [code] new Giki({ 'div' : 'giki_div', 'text_area' : 'giki_ta', 'parser' : 'bbcode.js' });[/code] Et voila! [b]Adding new parsers[/b] Giki is designed with extensibility in mind. Writing new parsers is very simple. New parsers are written as web workers (they will also work in older browsers thanks to gworker) and therefore only need to implement the onmessage function, and use the postMessage function to return the parsed data. [code] self.onmessage = function(event) { var data = event.data; // ...... // Do some parsing // ....... self.postMessage(data); };[/code]
